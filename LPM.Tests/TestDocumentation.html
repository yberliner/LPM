LPM – Unit Test Documentation

Project: LPM (Dror Center – LifePower)
Database: lifepower.db (SQLite)
Test framework: xUnit (.NET 8)
Isolation strategy: Each test class creates a fresh temporary SQLite file; the file is deleted after the test class is disposed.
Date prepared: March 2026

This document describes every unit test in the LPM.Tests project, organized by the service under test. Tests are written in C# using xUnit and test the data-access layer against an in-memory-isolated SQLite database – no live lifepower.db is ever touched during test runs.

1. Test Infrastructure

1.1 TestDbHelper

Creates a disposable SQLite database at a unique temporary path and populates the full schema (Persons, Auditors, CaseSupervisors, PCs, Sessions, CsReviews, CsWorkLog, Payments, StaffPcList, MiscCharge, Students, Grades, Users, Roles, UserRoles). Seed data: three Grade rows (BA, MA, PHD).

1.2 TestConfig

Builds a minimal IConfiguration that sets Database:Path to the temporary file created by TestDbHelper, allowing every service constructor to be instantiated without needing a running ASP.NET host.

7. How to Run the Tests

  1. Open Visual Studio 2022.
  2. Add the LPM.Tests project to the solution: File → Add → Existing Project → select LPM\LPM.Tests\LPM.Tests.csproj.
  3. Build the solution (Ctrl+Shift+B).
  4. Open the Test Explorer (Test → Test Explorer).
  5. Click Run All Tests.

Alternatively, from the Developer Command Prompt or terminal:

cd "C:\Dev\Yaniv-Projects\Dror center\LPM"
dotnet test LPM.Tests/LPM.Tests.csproj --verbosity normal

Each test creates and destroys its own temporary SQLite file in %TEMP%. No network connections, live database, or external services are required.
